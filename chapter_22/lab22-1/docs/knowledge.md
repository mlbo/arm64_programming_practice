# NEON 基础知识

## NEON 技术概述

NEON 是 ARM 处理器的 128-bit SIMD (Single Instruction Multiple Data) 扩展，属于 ARMv8-A 架构的标准组件。

### NEON 寄存器

ARMv8-A 提供 32 个 128-bit NEON 寄存器 (V0-V31)：

```
+------------------------------------------------------------------+
|  V0  [127:0] - 128 bits                                          |
+------------------------------------------------------------------+
|  V1  [127:0] - 128 bits                                          |
+------------------------------------------------------------------+
|  ...                                                              |
+------------------------------------------------------------------+
|  V31 [127:0] - 128 bits                                          |
+------------------------------------------------------------------+
```

每个 128-bit 寄存器可以视为：
- 16 个 8-bit 元素 (uint8x16_t)
- 8 个 16-bit 元素 (uint16x8_t)
- 4 个 32-bit 元素 (uint32x4_t, float32x4_t)
- 2 个 64-bit 元素 (uint64x2_t, float64x2_t)

### 向量类型命名规则

```cpp
uint8x16_t   // 16 个 uint8_t  元素
int8x16_t    // 16 个 int8_t   元素
float32x4_t  // 4  个 float    元素
uint8x16x3_t // 3  组 uint8x16_t (交错访问用)
```

## 交错加载/存储

### RGB 图像内存布局

RGB24 格式中，每个像素的 R, G, B 分量连续存储：

```
内存地址:  0   1   2   3   4   5   6   7   8   ...
像素:     [R0][G0][B0][R1][G1][B1][R2][G2][B2]...
```

### vld3q_u8 指令

`vld3q_u8` 一次加载 48 字节 (16 像素)，分离成 3 个向量：

```cpp
uint8x16x3_t rgb = vld3q_u8(src);
// rgb.val[0] = R0, R1, R2, ..., R15  (所有 R 分量)
// rgb.val[1] = G0, G1, G2, ..., G15  (所有 G 分量)
// rgb.val[2] = B0, B1, B2, ..., B15  (所有 B 分量)
```

### vst3q_u8 指令

`vst3q_u8` 将 3 个向量交错存储：

```cpp
vst3q_u8(dst, bgr);
// 存储: [B0][G0][R0][B1][G1][R1]...
```

## NEON Intrinsics 分类

| 类别 | 示例 | 说明 |
|------|------|------|
| 加载 | vld1q_u8, vld3q_u8 | 从内存加载向量 |
| 存储 | vst1q_u8, vst3q_u8 | 向量存储到内存 |
| 算术 | vaddq_u8, vmulq_f32 | 向量算术运算 |
| 移动 | vmovq_n_u8 | 创建常量向量 |
| 重排 | vrev64q_u8 | 向量元素重排 |

## 编译选项

```makefile
# 启用 NEON (ARMv8-A 默认支持)
CXXFLAGS = -march=armv8-a+simd

# 优化级别
CXXFLAGS += -O2 -ftree-vectorize
```
