# 中断处理与栈帧 - 练习题

## 练习 1: 理解栈帧

**问题：** 为什么 `pt_regs` 结构体需要保存 31 个通用寄存器 (x0~x30)？

**提示：** 考虑中断是异步发生的，被中断的代码可能正在使用任何寄存器。

---

## 练习 2: 栈帧大小计算

**问题：** 计算以下结构体的大小：
```cpp
struct PtRegs {
    u64 regs[31];   // x0~x30
    u64 sp;
    u64 pc;
    u64 pstate;
};
```

**要求：**
1. 说明每个字段的大小和偏移
2. 解释为什么总大小必须是 16 的倍数

## 练习 4: 实现同步异常处理

**任务：** 在 `entry.S` 中实现一个同步异常处理函数 `el1_sync`，要求：
1. 保存寄存器状态
2. 读取 `ESR_EL1` 寄存器获取异常原因
3. 调用 C 函数 `do_sync_exception(PtRegs* regs, u32 esr)`
4. 恢复寄存器状态并返回

**框架代码：**
```asm
el1_sync:
    kernel_entry
    // TODO: 实现
    kernel_exit
```

## 练习 8: 调试技巧

**任务：** 编写一个函数 `dump_pt_regs(const PtRegs* regs)`，打印所有保存的寄存器值。

**要求：**
```cpp
void dump_pt_regs(const PtRegs* regs) {
    printk("=== PT_REGS ===\n");
    // 打印 x0~x30
    // 打印 sp, pc, pstate
}
```

**预期输出：**
```
=== PT_REGS ===
x0: 0x0000000000000001  x1: 0x0000000000000002
...
x30: 0x0000000000080100
sp: 0x0000000000080000
pc: 0x0000000000090000
pstate: 0x00000000000003C5
```
