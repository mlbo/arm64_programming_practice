# 知识点：BL 指令与返回地址保护

## 1. BL 指令

```asm
bl function    // 跳转到 function，并将返回地址保存到 x30 (LR)
```

**重要**: BL 指令会将下一条指令的地址写入 x30 寄存器。

## 2. 返回地址丢失问题

当函数 A 调用函数 B，函数 B 又调用函数 C 时：

```
A 调用 B:  x30 = A 的返回地址
B 调用 C:  x30 = B 的返回地址  <- 覆盖了 A 的返回地址！
B 返回:    跳转到 B 的返回地址
A 返回:    x30 已经丢失，程序跑飞！
```

## 3. 解决方案

### 方案一：保存 x30 到栈

```asm
bl_test:
    stp x29, x30, [sp, #-16]!  // 保存 FP 和 LR
    mov x0, 1
    mov x1, 3
    bl csel_test
    ldp x29, x30, [sp], #16    // 恢复 FP 和 LR
    ret
```

### 方案二：使用 BLR 和立即返回

```asm
bl_test:
    mov x0, 1
    mov x1, 3
    bl csel_test
    // 直接返回，不保存 x30（仅适用于叶子函数）
```

## 4. 栈帧布局

```
高地址
+------------------+
| 返回地址 (x30)   |
+------------------+
| 保存的 FP (x29)  |
+------------------+ <- FP
| 局部变量         |
+------------------+
低地址
```
