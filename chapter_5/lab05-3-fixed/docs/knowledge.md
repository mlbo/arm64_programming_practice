# 知识点：返回地址保护

## 1. 为什么需要保护 x30？

BL 指令会将返回地址写入 x30，如果函数中再次调用 BL，x30 会被覆盖。

## 2. ARM64 调用约定

### 调用者保存寄存器
- x0-x7: 参数/返回值
- x8-x17: 临时寄存器
- x18: 平台寄存器

### 被调用者保存寄存器
- x19-x28: 必须保存
- x29 (FP): 帧指针
- x30 (LR): 返回地址

## 3. 标准栈帧建立

```asm
func:
    stp x29, x30, [sp, #-16]!  // 1. 保存 FP 和 LR
    mov x29, sp                // 2. 设置新的 FP
    sub sp, sp, #N             // 3. 分配局部变量空间

    // 函数体

    mov sp, x29                // 4. 恢复 SP
    ldp x29, x30, [sp], #16    // 5. 恢复 FP 和 LR
    ret
```

## 4. 叶子函数

如果一个函数不调用其他函数（叶子函数），可以不保存 x30。

```asm
leaf_func:
    // 不需要保存 x30
    add x0, x0, x1
    ret
```
